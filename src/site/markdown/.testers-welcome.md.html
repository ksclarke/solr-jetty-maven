Testers Welcome!

There are many ways to contribute to an open source project. One way is to use the project and report back with any problems you face. This helps developers improve the project by surfacing bugs and areas for improvement (either in the documentation, the user interface, or in the code itself). You don't have to be a programmer to download and run a project. In fact, testing out the project as someone who doesn't program will sometime surface issues that programmers are likely to overlook.

Report an Issue

If you do discover an area of the project that could use some improvement (or just plain fixing), please file a ticket. This will alert the developers to the issue. When reporting an issue, try to be as specific as you can. It's helpful if you can provide a series of reproducible steps that developers can follow to discover the issue you're reporting. Any issue report, though, is useful.

Coders Welcome, Too

If you are a tester who has programming experience, patches are of course also welcome. See the Coders Welcome page for more details about how to contribute in this way.
